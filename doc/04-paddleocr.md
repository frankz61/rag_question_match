# PaddleOCR 图像识别微服务——架构汇报

---

## 一、模块定位

本模块是整个题库处理链路的**图像文字识别基础能力层**，负责将图片题目转换为可处理的文本，打通从"图片输入"到"语义检索"的入口。

基于 **FastAPI + PaddleOCR** 封装为独立 HTTP 微服务，下游直接对接 BGE-M3 Embedding → Milvus 检索链路。

---

## 二、核心功能

- **图片 OCR 识别**：支持 png/jpg/jpeg/bmp/tiff 格式，中文识别含倾斜校正（`use_angle_cls=True`）
- **异步非阻塞**：使用 `run_in_threadpool` 将 OCR 推理投入线程池，主事件循环不被阻塞
- **线程安全缓存**：通过 `threading.local()` 实现线程级 Reader 复用，彻底消除每次请求重新加载模型的开销
- **硬件自适应**：运行时自动探测 CUDA，有 GPU 用 GPU，无则自动降级 CPU，开发机与生产环境同一套代码
- **PDF 支持预留**：底层逻辑已实现，接口可按需快速上线
- **健康检查**：`GET /health`，供容器编排平台探活
